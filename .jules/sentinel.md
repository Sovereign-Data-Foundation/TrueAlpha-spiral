## 2026-03-30 - [Command Injection via Inadequate Command Restriction]
**Vulnerability:** Command injection/arbitrary command execution in `tas_pythonetics/src/tas_pythonetics/git_safety.py`. `GitActionGuard.authorize_command` failed to verify that the command being checked actually started with "git".
**Learning:** Security guards designed to filter specific harmful arguments (like `rebase` or `--force`) must also strictly enforce the root command identity, otherwise they inadvertently allow entirely different executables to bypass the filters.
**Prevention:** Always validate the root command (the first token) against a strict allowlist before applying argument-specific filters when performing command execution filtering.
## 2026-03-31 - [Command Injection via File Redirection and Backgrounding Bypass]
**Vulnerability:** Arbitrary file overwrite and command execution bypass in `codex_tas_runner.py` via shell redirection (`>`, `>>`, `<`) and background operators (`&`).
**Learning:** Security tokenizers that split command strings using a limited set of operators (e.g., `&&`, `||`, `;`) often ignore other execution flow operators like redirection and backgrounding. This allows attackers to redirect output to arbitrary files or bypass validation checks entirely because the validator erroneously treats the subsequent file names or commands as valid contiguous arguments to the initial allowed command.
**Prevention:** When parsing raw commands to enforce an allowlist, ensure the token splitting regular expression accounts for all bash control operators, including redirection and background execution, and explicitly reject them if unsupported.
